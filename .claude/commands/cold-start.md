name: cold-start
description: Session initialization — re-establish context from documentation
---

Starting a new session. Read the following in order and confirm understanding:

1. Read DEVPLAN.md — focus on Cold Start Summary and Current Status
2. Read the current phase's section in DEVPLAN for active work context
3. If Current Status shows Blocked/Broken, read the relevant DEVLOG
   entries to understand the issue

Then report:
- Current phase and step
- What was being worked on
- Any blockers or open issues
- What the next action should be

**If autonomous:**
In autonomous mode, cold-start is handled by the iteration protocol
in CLAUDE.md. If invoked directly, proceed to determine next action
from DEVPLAN without waiting.

**If not autonomous** (default):
Do not start working until I confirm the direction.
